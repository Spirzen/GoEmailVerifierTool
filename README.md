# GoEmailVerifierTool

CLI-утилита на Go для проверки email-адресов: синтаксис, DNS (MX) и SMTP (RCPT TO).

## Возможности

- Проверка формата адреса
- Поиск и сортировка MX-записей домена
- SMTP-проверка существования ящика через `RCPT TO` (без отправки письма)
- Настраиваемый таймаут DNS/SMTP операций
- Режим просмотра MX-записей без SMTP-проверки

## Требования

- Go 1.22+
- Исходящий доступ к DNS (UDP 53) и SMTP (TCP 25)

> **Примечание:** многие провайдеры и корпоративные сети блокируют исходящий порт 25. В таких условиях SMTP-проверка может быть недоступна, хотя DNS (MX) продолжит работать.

## Установка

```bash
git clone https://github.com/timta/GoEmailVerifierTool.git
cd GoEmailVerifierTool
go build -o email-verifier .
```

Или установить напрямую (бинарник будет называться `GoEmailVerifierTool`):

```bash
go install github.com/timta/GoEmailVerifierTool@latest
```

## Использование

```bash
# Проверить email
email-verifier user@gmail.com

# Показать MX-записи домена
email-verifier -mx google.com
email-verifier -mx user@gmail.com

# Увеличить таймаут
email-verifier -timeout 30s user@example.com
```

### Флаги

| Флаг | По умолчанию | Описание |
|------|--------------|----------|
| `-timeout` | `15s` | Таймаут DNS/SMTP операций |
| `-mx` | `false` | Только показать MX-записи, без SMTP-проверки |

### Пример вывода

```
Email:  user@gmail.com
Формат: корректный
Domain: gmail.com
MX:     gmail-smtp-in.l.google.com
SMTP:   вероятно существует (сервер принял RCPT TO)
```

### Коды выхода

| Код | Условие |
|-----|---------|
| `0` | Адрес корректен и SMTP-проверка не опровергла существование |
| `1` | Некорректный формат, ошибка DNS/SMTP или адрес не существует |

## Как это работает

1. **Синтаксис** — проверка формата адреса регулярным выражением.
2. **DNS** — `LookupMX` для домена, MX-серверы сортируются по приоритету.
3. **SMTP** — подключение к MX на порту 25, `EHLO`, `MAIL FROM:<>` и `RCPT TO:<email>`. Письмо не отправляется.

При недоступности первого MX- сервера утилита пробует следующий по приоритету.

## Ограничения

SMTP-проверка существования ящика **не даёт 100% гарантии**:

- **Catch-all** — домен принимает любой адрес; `RCPT TO` вернёт успех даже для несуществующего ящика.
- **Greylisting** — временный отказ может быть ошибочно истолкован как «не существует».
- **Блокировка порта 25** — SMTP-этап недоступен из вашей сети.
- **Rate limiting** — почтовые серверы могут отклонять частые проверки.
- **Политики провайдеров** — Gmail, Outlook и другие крупные сервисы часто не раскрывают реальное состояние ящина через RCPT TO.

Результат `SMTP: вероятно существует` означает, что сервер **принял** адрес на этапе `RCPT TO`, а не подтверждает доставку письма.

## Ответственное использование

Проверяйте только адреса, на обработку которых у вас есть законное основание. Массовая автоматическая проверка может нарушать политики почтовых провайдеров и привести к блокировке вашего IP. Используйте утилиту осознанно и в соответствии с применимым законодательством.
